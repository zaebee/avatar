## Цель

Система для отправки контента (фото + текст) из условий ограниченного интернета (деревня) и публикации в Instagram через агента asi:one.

## Компоненты

### 1. Web-encryptor (client-side)

**Назначение:** Шифрование контента перед отправкой в email.

**Интерфейс:**
- HTML-страница (одна)
- Вход: текст поста, загрузка фото (1-2 шт)
- Пароль: shared secret (фиксированный, известен пользователю)
- Выход: зашифрованная строка (base64)

**Технология:** vanilla JS + Web Crypto API (AES-256-GCM)

**Сжатие:** JPEG до 1MB перед шифрованием (canvas API)

### 2. IMAP-poller Cloud Function

**Назначение:** Периодическая проверка ящика на новые письма.

**Триггер:** по расписанию (каждые 5 минут)  
**Протокол:** IMAP (imap.yandex.ru)  
**Поведение:**
- Подключается к ящику с credentials из env
- Ищет письма с вложениями
- Для каждого письма:
  1. Извлекает тело (зашифрованный текст)
  2. Скачивает вложения → Object Storage
  3. Декодирует base64 → дешифрует (AES-256-GCM)
  4. Публикует в очередь
  5. Перемещает письмо в архив

### 3. Message Queue

**Назначение:** Гарантированная доставка сообщений.

**Структура сообщения:**
```json
{
  "text": "пост из деревни",
  "images": ["https://storage.yandex.net/photos/xxx.jpg"],
  "timestamp": "2026-04-11T12:00:00Z",
  "from": "user@yandex.ru"
}
```

**Retry-логика:** 3 попытки с экспоненциальной задержкой.

### 4. asi:one интеграция

**Назначение:** Формирование и публикация поста в Instagram.

**ASI:ONE:**
- Endpoint: `https://api.asi1.ai/v1/chat/completions`
- Модель: `asi1`
- Инструмент: существующий instagram-инструмент агента
- Аккаунт: `@zaebuntu`

**Поведение:**
1. Polling очереди (каждые 1 минута)
2. Формирование промпта: "Опубликуй в Instagram..."
3. Вызов asi:one API
4. asi:one выполняет instagram-инструмент
5. Удаление сообщения из очереди

## Безопасность

### Шифрование

- **Алгоритм:** AES-256-GCM
- **Ключ:** PBKDF2 (100k итераций, SHA-256) из shared secret
- **IV:** Уникальный 96-bit random IV для каждого шифрования
- **Формат:** base64(IV + ciphertext + auth_tag)
- **Пароль пользователя:** Не передаётся в письме

### Credentials

- IMAP: хранятся в Yandex Cloud Secrets
- ASI:ONE API key: хранится в Yandex Cloud Secrets

## Риски и митигации

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| IMAP недоступен | Низкая | Retry + логирование в Cloud Logging |
| Шифрование не работает | Средняя | Unit-тесты на шифрование |
| asi:one недоступен | Низкая | Очередь с retry (до 3 раз) |
| Фото слишком большое | Средняя | Сжатие на клиенте до 1MB перед шифрованием |
| Shared secret скомпрометирован | Низкая | Ротация ключа через env |

## Тестирование

### E2E-сценарий

1. Пользователь офлайн в деревне
2. Открывает web-encryptor
3. Вводит текст, загружает фото
4. Нажимает "Зашифровать" → копирует зашифрованную строку
5. Открывает Яндекс.Почту → отправляет письмо с зашифрованной строкой в теле
6. Когда появляется инет → письмо уходит
7. Cloud Function (через 5 мин) обрабатывает письмо
8. asi:one публикует пост в Instagram

### Unit-тесты

- Шифрование/дешифрование (проверка AES-256-GCM)
- Cloud Function (Mock IMAP + MQ)
- Обработка ошибок (невалидный пароль, большое фото)
